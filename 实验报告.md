## 军棋陆战棋（Junqi）

### 一、项目概述

本项目实现了一款基于 CNVCS 规则的军棋陆战棋桌面应用，支持人机对战、网络对战、布阵管理、复盘查看等功能。采用 Rust 语言开发，GUI 基于 egui/eframe 框架。

### 二、系统架构

项目采用 Cargo Workspace 组织，分为 6 个子 crate：

```
junqi/
├── crates/
│   ├── junqi-core/       # 核心引擎：棋盘、棋子、规则、对局状态机
│   ├── junqi-ai/         # AI 引擎：局面评估、Minimax 搜索、MCTS
│   ├── junqi-net/        # 网络层：TCP 协议、房间管理、对局同步
│   ├── junqi-storage/    # 持久化层：布阵/复盘存储、用户设置
│   ├── junqi-rendezvous/ # 中继服务器：房间注册与消息转发
│   └── junqi-app/        # GUI 前端：eframe 界面、棋盘渲染、屏幕路由
```

#### 3.1 junqi-core —— 核心引擎

**数据结构**：
- `PieceKind`：12 种棋子枚举（司令、军长、…、军旗），每种有等级、数量、移动能力
- `Color`：红/蓝双方
- `Position`：棋盘坐标 (row 0-11, col 0-4)
- `Board`：12×5 棋盘，预计算公路/铁路邻接表
- `Game`：对局状态机（NotStarted → Deploying → Playing → Finished）

**核心算法**：
- 无，都是非常 trivial 的东西。

#### 3.2 junqi-ai —— AI 引擎

提供 4 档难度：

| 难度 | 算法 | 搜索深度 | 特点 |
|------|------|----------|------|
| 初级 | Minimax | 2 | 30% 随机 |
| 中级 | Minimax + Alpha-Beta | 4 | 走法排序 |
| 高级 | Minimax + Alpha-Beta | 6 | 迭代加深 |
| 专家 | MCTS | 5000 模拟 | 信息不完全博弈优化 |

**评估函数**维度：棋子面值、位置加成、行营安全、机动性、威胁度。

#### 3.4 junqi-app —— GUI 前端

- 主菜单 → 难度选择 → 布阵界面 → 对战界面

### 四、关键技术与实现细节

#### 4.1 棋盘连通性

棋盘有公路和铁路两种移动方式。公路仅走 1 步，铁路可滑行（工兵可沿铁路网 BFS 转弯，非工兵仅直线）。在 `Board::new()` 时预计算所有邻接关系存入 `road_adj` 和 `rail_adj`，走法生成时直接查表。

#### 4.2 暗棋机制

CNVCS 规则下，初始时对方棋子不可见。碰撞时双方棋子互相揭示（`execute_move()` 中在战力判定前调用 `reveal()`）。复盘回放时同样通过 `execute_move()` 复盘，自动还原揭示状态。

#### 4.3 迭代加深 Alpha-Beta 搜索

从深度 1 开始逐步加深至 max_depth，每次迭代利用上一次的最佳走法优化排序。配合时间限制（`time_limit_ms`），可在时限内返回当前最优解。

#### 4.4 MCTS 搜索

实现四阶段：Selection（UCT 选择）→ Determinization（暗棋随机化）→ Rollout（随机模拟）→ Backpropagation（反向传播）。根节点预展开所有合法走法，每轮模拟后更新整条路径的胜率。

#### 4.5 棋盘翻转

为提升用户体验，玩家阵地始终显示在棋盘下方。`BoardWidget::flip_board` 控制 Y 轴翻转，红方玩家棋盘上下颠倒显示，蓝方不变。

### 五、运行方式

```bash
# 编译运行 GUI 客户端
cargo run -p junqi-app --release

# 启动中继服务器（网络对战需要）
cargo run -p junqi-rendezvous --release

# 生成 API 文档
cargo doc --no-deps --document-private-items --open
```

### 六、收获与体会

1. Rust 的调试和修改真的比别的语言快很多，我认为熟练掌握后其开发速度完全不比别的语言差，而且代码量由于有模式匹配等功能，也比 C++ 要少，运行效率也不输。

2. Cargo 的项目管理非常好用，项目的结构骨架非常清晰。

3. Doc attr 非常友好，点开网页链接后成就感满满。

### 八、项目上传指南

#### 方式一：GitHub 上传（推荐）

```bash
# 1. 在项目根目录初始化 Git
cd junqi
git init

# 2. 添加所有文件（.gitignore 已配置好，会自动排除 target/）
git add .

# 3. 提交
git commit -m "军棋陆战棋 Rust 项目"

# 4. 在 GitHub 网页上创建新仓库（不要勾选 README/LICENSE/.gitignore）

# 5. 关联远程仓库并推送
git remote add origin https://github.com/<你的用户名>/junqi.git
git branch -M main
git push -u origin main
```

#### 方式二：直接打包

```bash
# 打包为 zip（Windows PowerShell）
Compress-Archive -Path * -DestinationPath junqi.zip
```
